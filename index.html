Featured Project

Atlas

A private AI workflow system I built to explore how agents use tools, manage context, run evals, and recover from mistakes around sensitive personal data.

Private system / public lessons

Agent workflows, evals, tools, and recovery paths.

Atlas is the clearest example of the product problems I like: turning model capability into dependable workflows, designing tool surfaces agents can actually use, and building enough evaluation and recovery scaffolding to trust the system over time.

View Atlas project

What I learned

  • Tool-use claims are not evidence.
  • General tools can make agents less reliable.
  • Evals need to test real workflows, not demos.
  • AI systems that write data need recovery paths.

Notes

Short notes from building and studying AI products.

When an agent says it used a tool, that is not evidence

A lesson from an agent that cited commands, produced plausible answers, and still routed around the system it was supposed to use.

AI agents debug symptoms before systems

How a stale workspace, a tiny taxonomy bug, and mismatched queries turned into a very convincing wrong theory.

When more powerful tools make agents worse

Replacing narrow tools with a generalized scenario engine made Atlas more capable, but also easier for the agent to misuse.

Evals should test the workflow, not the demo

The agent can know the right rule and still fail because the eval missed screenshots, session state, file writes, or side effects.

If an AI can write data, it needs a recovery path

Confirm-before-commit, undo tools, audit trails, and out-of-band controls are product features, not implementation details.

Product questions

Questions I keep coming back to.

  • What should an AI product refuse to automate?
  • When does adding flexibility make a system harder to trust?
  • How do you design for recovery when the model is wrong?
  • What should the user control, and what should the system infer?
  • How do you evaluate an agent when the task itself is ambiguous?
  • What makes a workflow feel trustworthy enough to become a habit?
